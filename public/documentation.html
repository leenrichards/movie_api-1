Shrek Movies API

This is going to be a web application that will provide users with access to information about Shrek movies.
Users will be able to sign up, update their personal information, and create a list of their favorite Shrek franchise parts.

Technical Requirements

  • The API must be a Node.js and Express application;
  • The API must use REST architecture, with URL endpoints corresponding to the data operations listed above;
  • The API must use at least three middleware modules, such as the body-parser package for reading data from requests and morgan for logging;
  • The API must use a “package.json” file;
  • The database must be built using MongoDB;
  • The business logic must be modeled with Mongoose;
  • The API must provide movie information in JSON format;
  • The JavaScript code must be error-free;
  • The API must be tested in Postman;
  • The API must include user authentication and authorization code;
  • The API must include data validation logic;
  • The API must meet data security regulations;
  • The API source code must be deployed to a publicly accessible platform like GitHub;
  • The API must be deployed to Heroku.

Essential Features

Request URL HTTP Method Response
Home page of myFlix App / GET Message: "Welcome to myFlix App!"
Return a list of ALL movies to the user GET /movies Return array of all JSON objects in the database
Return data about one movie by title GET /movies/:title Return JSON object with data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title
Example response
{ id: "...", title: "...", description: "...", genre: "...", director: "...", imagePath: "...", featured: "..." }
Return data about genre GET /genres/:name Return JSON object with data about genre (description) by name/title (e.g., “Thriller”)
Example response
{ title: "...", genre: "...", }
Return data about a director GET /directors/:name Return JSON object with data about director (bio, birth year, death year) by name
Example response
{ director: "...", name: "...", bio: "...", birth: "...", death: "..." }
Return a list of ALL users GET /users Return array of all JSON objects in the database
Return data about a user by username GET /users/:username Return JSON object with data about a user by name
Allow new users to register POST /users Create JSON object with data about a new user.
Example response
{ id: "...", username: "...", password: "...", email: "...", birthday: "...", favoriteMovies: "..." }
Allow users to update their user info PUT /users/:username Update JSON object with user data.
Example
{ id: "...", username: "...", password: "...", email: "...", birthday: "...", favoriteMovies: "..." }
Allow users to add a movie to their list of favorites POST /users/:username/movies/:movieID Create JSON object with data about a user's list of favorite movies. Message: "Movie was added to favorites"
Allow users to remove a movie to their list of favorites. DELETE /users/:username/movies/:movieID JSON object with data about a user's list of favorite movies. Message: "Movie was deleted"
Allow existing users to deregister DELETE /users/:username Delete user's account. Message: "Your account was successfully deleted"
